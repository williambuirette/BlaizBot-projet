# Glossaire

> Définitions des termes techniques utilisés dans cet exposé

---

## A

**Abstract** : Résumé d'un document académique, généralement en anglais.

**Agile** : Méthodologie de développement logiciel basée sur des itérations courtes et l'adaptation continue.

**API (Application Programming Interface)** : Interface permettant à des logiciels de communiquer entre eux. Dans BlaizBot, les API Routes Next.js servent à connecter le frontend à la base de données et aux services IA.

**App Router** : Système de routage de Next.js 13+ basé sur l'arborescence des dossiers dans `app/`.

**Assignment** : Assignation d'un cours ou exercice à un élève ou une classe, avec éventuelle date limite.

---

## B

**Backend** : Partie serveur d'une application, qui gère la logique métier, les données et les API.

**Backlog** : Liste priorisée des tâches à réaliser dans un projet.

**Brainstorming** : Technique de créativité pour générer des idées sans jugement initial.

---

## C

**Chat IA** : Interface conversationnelle permettant d'interagir avec un modèle de langage.

**Claude** : Famille de modèles de langage développée par Anthropic (Claude Sonnet, Claude Opus).

**Commit** : Enregistrement d'une modification dans un système de gestion de versions (Git). Chaque commit a un message décrivant les changements.

**Composant** : Élément réutilisable d'interface utilisateur en React (ex: bouton, carte, formulaire).

**CRUD** : Create, Read, Update, Delete – les quatre opérations de base sur les données.

---

## D

**Dashboard** : Tableau de bord regroupant des indicateurs clés et informations importantes.

**Déploiement** : Mise en ligne d'une application sur un serveur accessible via Internet.

**Design System** : Ensemble de règles et composants visuels garantissant la cohérence d'une interface.

**Dette technique** : Accumulation de choix techniques imparfaits qui compliquent la maintenance future.

---

## E

**ESLint** : Outil d'analyse statique de code JavaScript/TypeScript qui détecte les erreurs et problèmes de style.

---

## F

**Frontend** : Partie visible d'une application, l'interface utilisateur avec laquelle l'utilisateur interagit.

**Full-stack** : Se dit d'une application ou d'un développeur qui gère à la fois le frontend et le backend.

---

## G

**Gemini** : Famille de modèles de langage développée par Google DeepMind (Gemini 2.0 Flash utilisé dans BlaizBot).

**Git** : Système de gestion de versions décentralisé permettant de suivre l'historique des modifications.

**GitHub** : Plateforme d'hébergement de code utilisant Git, avec fonctionnalités de collaboration.

**GitHub Copilot** : Assistant IA de codage intégré dans VS Code, basé sur OpenAI Codex.

**GPT (Generative Pre-trained Transformer)** : Architecture de modèle de langage développée par OpenAI (GPT-4, GPT-4o).

---

## H

**Hallucination** : Génération de contenu faux ou inventé par une IA (ex: bibliothèques inexistantes, fonctions imaginaires).

---

## I

**IDE (Integrated Development Environment)** : Environnement de développement intégré (ex: VS Code).

**Itération** : Cycle court de développement produisant une version améliorée du produit.

---

## J

**JWT (JSON Web Token)** : Format de token utilisé pour l'authentification sécurisée.

---

## K

**Kickoff** : Prompt de lancement initial d'une phase de développement.

**KPI (Key Performance Indicator)** : Indicateur clé de performance mesurant un aspect important (ex: taux de réussite, progression).

---

## L

**Layout** : Mise en page commune partagée entre plusieurs pages (ex: sidebar + header).

**LLM (Large Language Model)** : Grand modèle de langage entraîné sur d'énormes corpus de texte (GPT, Claude, Gemini).

**Linter** : Outil d'analyse de code détectant les erreurs et problèmes de style (ESLint).

**LMS (Learning Management System)** : Plateforme de gestion de l'apprentissage (ex: Moodle, Canvas).

---

## M

**Markdown** : Langage de balisage léger pour formater du texte (fichiers `.md`).

**Middleware** : Logiciel intermédiaire s'exécutant entre la requête et la réponse (ex: vérification d'authentification).

**Migration** : Modification structurelle de la base de données (ajout/suppression de tables ou colonnes).

**MVP (Minimum Viable Product)** : Version minimale fonctionnelle d'un produit, avec les fonctionnalités essentielles.

---

## N

**Next.js** : Framework React pour le développement web full-stack avec rendu côté serveur.

**NextAuth** : Bibliothèque d'authentification pour Next.js (v5 utilisée dans BlaizBot).

**Node.js** : Environnement d'exécution JavaScript côté serveur.

---

## O

**ORM (Object-Relational Mapping)** : Outil de correspondance entre objets et base de données relationnelle (ex: Prisma).

---

## P

**PostgreSQL** : Système de gestion de base de données relationnelle open-source.

**Prettier** : Outil de formatage automatique de code.

**Prisma** : ORM moderne pour Node.js et TypeScript, avec génération de types automatique.

**Prompt** : Instruction ou consigne donnée à une IA générative pour obtenir une réponse.

**Prompt engineering** : Art de formuler des prompts efficaces pour obtenir les meilleurs résultats d'une IA.

**Prompt stack** : Empilement de plusieurs couches de prompts (système, contexte, utilisateur) pour guider l'IA.

**Prototype** : Version préliminaire d'une application, souvent simplifiée, pour tester un concept.

---

## R

**RAG (Retrieval-Augmented Generation)** : Technique combinant recherche d'informations et génération de texte par l'IA.

**React** : Bibliothèque JavaScript pour créer des interfaces utilisateur basées sur des composants.

**RBAC (Role-Based Access Control)** : Contrôle d'accès basé sur les rôles utilisateurs.

**Refactoring** : Restructuration du code pour améliorer sa lisibilité ou maintenabilité, sans changer son comportement.

**Route** : URL permettant d'accéder à une page ou une API (ex: `/student/dashboard`, `/api/courses`).

---

## S

**Seed** : Jeu de données initial inséré dans la base de données pour les tests.

**shadcn/ui** : Collection de composants React réutilisables et accessibles, basée sur Radix UI.

**Stack technique** : Ensemble des technologies utilisées dans un projet (langages, frameworks, outils).

**Streaming** : Envoi progressif de données (ex: réponse de l'IA affichée mot par mot).

---

## T

**Tailwind CSS** : Framework CSS utility-first pour styliser rapidement des interfaces.

**TODO** : Liste de tâches à réaliser (fichier `TODO.md` dans un projet).

**Token** : Unité de texte pour un LLM (un mot ≈ 1,3 tokens en français).

**TypeScript** : Langage de programmation ajoutant le typage statique à JavaScript.

---

## U

**UI (User Interface)** : Interface utilisateur, partie visible d'une application.

**UX (User Experience)** : Expérience utilisateur, qualité de l'interaction avec une application.

---

## V

**Vercel** : Plateforme de déploiement cloud optimisée pour Next.js.

**Vibe Coding** : Méthodologie de développement assistée par IA, où le développeur guide l'IA via des prompts itératifs.

**VS Code (Visual Studio Code)** : Éditeur de code open-source développé par Microsoft.

---

## W

**Wireframe** : Maquette simplifiée d'une interface, montrant la structure et les éléments principaux sans design détaillé.

**Workflow** : Flux de travail, séquence d'étapes pour réaliser une tâche.
