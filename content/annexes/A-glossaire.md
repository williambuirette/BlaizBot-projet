# Annexe A - Glossaire

> Définitions des termes techniques et acronymes utilisés dans cet exposé.

---

## A. Termes généraux

| Terme | Définition |
| :--- | :--- |
| **API** | Application Programming Interface - Interface permettant à deux applications de communiquer |
| **Backend** | Partie serveur d'une application (logique métier, BDD) |
| **Boilerplate** | Code répétitif standard, souvent généré automatiquement |
| **CRUD** | Create, Read, Update, Delete - Les 4 opérations de base sur les données |
| **Frontend** | Partie client d'une application (interface utilisateur) |
| **Full-stack** | Développement couvrant frontend et backend |
| **MVP** | Minimum Viable Product - Version minimale fonctionnelle d'un produit |

## B. Termes IA / Vibe Coding

| Terme | Définition |
| :--- | :--- |
| **Agent IA** | Entité IA spécialisée dans un domaine (code, docs, review) |
| **Contexte** | Informations fournies à l'IA pour comprendre la tâche |
| **Hallucination** | Génération de contenu incorrect ou inventé par l'IA |
| **LLM** | Large Language Model - Modèle de langage (GPT, Claude, etc.) |
| **Prompt** | Instruction donnée à une IA pour obtenir une réponse |
| **Prompt Engineering** | Art de formuler des prompts efficaces |
| **RAG** | Retrieval-Augmented Generation - IA enrichie par recherche documentaire |
| **Token** | Unité de texte (mot ou partie de mot) traitée par l'IA |
| **Vibe Coding** | Méthodologie de développement assisté par IA conversationnelle |

## C. Technologies du projet

| Terme | Définition |
| :--- | :--- |
| **GitHub Copilot** | Assistant IA de code intégré à VS Code (Microsoft/GitHub) |
| **Next.js** | Framework React pour applications web (Vercel) |
| **Node.js** | Runtime JavaScript côté serveur |
| **OpenAI** | Entreprise créatrice de GPT et des APIs d'IA |
| **PostgreSQL** | Système de gestion de base de données relationnelle |
| **Prisma** | ORM (Object-Relational Mapping) pour Node.js |
| **React** | Bibliothèque JavaScript pour interfaces utilisateur (Meta) |
| **Supabase** | Plateforme Backend-as-a-Service (BDD + Auth + Storage) |
| **Tailwind CSS** | Framework CSS utility-first |
| **TypeScript** | Superset de JavaScript avec typage statique |
| **Vercel** | Plateforme de déploiement pour applications web |
| **VS Code** | Visual Studio Code - Éditeur de code (Microsoft) |

## D. Termes méthodologiques

| Terme | Définition |
| :--- | :--- |
| **Agile** | Méthodologie de gestion de projet itérative |
| **CI/CD** | Continuous Integration/Continuous Deployment |
| **Commit** | Enregistrement d'une modification dans Git |
| **Conventional Commits** | Convention de nommage des commits (feat:, fix:, etc.) |
| **Definition of Done** | Critères pour considérer une tâche terminée |
| **Git** | Système de contrôle de version |
| **MoSCoW** | Must/Should/Could/Won't - Méthode de priorisation |
| **PR** | Pull Request - Demande de fusion de code |
| **Sprint** | Période de travail définie (généralement 1-2 semaines) |
| **User Story** | Description d'une fonctionnalité du point de vue utilisateur |

## E. Termes éducatifs

| Terme | Définition |
| :--- | :--- |
| **Apprentissage adaptatif** | Personnalisation du parcours selon le niveau de l'élève |
| **Feedback** | Retour d'information (positif/constructif) |
| **Gamification** | Utilisation de mécaniques de jeu dans l'éducation |
| **Hint-only** | Mode IA donnant des indices sans révéler la réponse |
| **Tuteur IA** | IA assistant l'apprentissage comme un tuteur humain |
